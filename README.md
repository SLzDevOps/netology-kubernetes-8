### 2. Расчет нод

##### Минимальная конфигурация
- **Количество нод:** 3
- **ОЗУ на ноду:** 24 ГБ (рекомендуется 32 ГБ)
- **CPU на ноду:** 12 ядер (рекомендуется 16 ядер)

##### Рекомендуемая конфигурация
- **Количество нод:** 4
- **ОЗУ на ноду:** 32 ГБ
- **CPU на ноду:** 16 ядер

### 3. Обоснование

1. **Отказоустойчивость:** 3 ноды обеспечивают распределение реплик БД и кеша.
2. **Запас на отказ:** При потере одной ноды, нагрузка перераспределяется на 2 оставшиеся.
3. **Служебные ресурсы:** 3 ГБ ОЗУ и 1 ядро CPU на ноду зарезервированы для системных процессов.
4. **Запас роста:** Рекомендуемые параметры (32 ГБ, 16 ядер) позволяют выдержать пиковые нагрузки и рост проекта.
